Speaking Pictures · CityMaker Documentation

Build cities.
Export anywhere.

CityMaker turns OpenStreetMap data into production-ready 3D cities inside Blender 5.x — then exports to Unreal Engine 5, Unity, or any engine via a JSON contract.

Blender 5.x Python 3.13 OSM · SRTM UE5 · Unity 28 Categories Phase 1 · v0.1

01 Quick Start

From zero to city in under five minutes.

  1. Install the plugin
    Blender Preferences ▸ Add-ons ▸ Install from Disk → pick citymaker-0.1.0.zip. Enable it. The CityMaker panel appears in the N-panel (press N in the 3D viewport).
  2. Load an OSM file
    In 1 · City Source, click Load .osm File and pick your downloaded OSM file, OR set your latitude/longitude and click Download from Overpass for a live download. Then click Import / Refresh OSM.
  3. Connect your assets
    In 2 · Asset Library, set the source to Folder and point it to a folder with building FBX/OBJ files in a buildings/ subfolder. Click Refresh Library.
  4. Generate everything
    In 3 · Generate City, click GENERATE ALL. In ~15 seconds you have terrain, roads, buildings, and optional signboards placed across the full city.
  5. Export
    In 4 · Export, choose your target engine and click Export. Run the generated import script inside your engine to finish.
Tip — No assets yet? Enable Create Dummy City in Section 1. CityMaker draws procedural hash-coloured geometry so you can explore the OSM layout without any library.

03 OSM Import

CityMaker reads OpenStreetMap data to understand what exists where.

📁 Load a local file
Use any .osm file exported from openstreetmap.org (Export → OpenStreetMap XML) or downloaded with Overpass Turbo.
🌐 Download live
Enter latitude, longitude, and area size (km²), then click Download from Overpass. CityMaker fetches the data directly from the OSM servers. Needs internet.

28-category classifier

CityMaker classifies every OSM element into one of 28 categories and places assets or geometry accordingly.

Category groupExamples
Buildingsresidential, commercial, industrial, historic, high_rise
Roads & pathsmotorway, residential, footway, cycleway, steps
Vegetationparks, forests, gardens, scrubland
Waterrivers, lakes, waterways, reservoirs
Publicschools, hospitals, places of worship, leisure
Infrastructurerailways, barriers, boundaries, parking
Coordinate mode Use Standard (fast) for cities under 50 km wide. Switch to High Accuracy (Transverse Mercator) for large or high-latitude areas where equirectangular projection introduces visible distortion.

04 Asset Library

Source modeWhen to useSetup
Folder FBX / GLB / OBJ / USD files on disk One subfolder per category: buildings/, signboards/, street_lamps/. Loose files count as buildings.
Tagged Collections Marketplace .blend (Superhive, Fab, Synty) Open the .blend, select objects, click Tag Selected As… → Buildings. Objects move to CityMaker/buildings collection.
Another .blend Assets in collections in a separate .blend Browse to the .blend. CityMaker links collections by category subfolder name.

Normalize Asset Size

Keep this ON (default). Marketplace and AI-generated assets come in wildly different scales — a building from one pack might be 0.45 m tall while another is 97 m. Normalization fits every asset to the OSM footprint without distorting proportions.

Nothing getting placed? If Normalize Asset Size is OFF and your assets span very different scales, CityMaker may skip every building (nothing fits). Turn ON Normalize Asset Size. The panel will warn you explicitly if placements = 0.

05 Terrain

🏔 Flat Default
A flat ground plane sized to the OSM bounds. No internet required. Right choice for stylised or game-optimised cities.
📡 Elevation (SRTM) Real heights
Samples real heights from OpenTopoData SRTM 30 m dataset. Needs internet. Set Grid for detail (higher = more API calls). Falls back to flat if the fetch fails.

Conform to Terrain

With Conform to Terrain on (default), buildings, signboards, and roads are dropped onto the terrain surface. Generate terrain before buildings — or use Drape on Terrain afterwards to snap everything that was already placed.

Order doesn't matter with Drape If you generated buildings first and terrain second, click Drape on Terrain — it snaps everything to the current terrain and rebuilds roads to follow the hills. Safe to run multiple times.

06 Roads

Roads are ribbon meshes built from OSM line features at real-world widths. Sidewalks are a second, slightly wider ribbon placed just below the road surface.

Road typeDefault widthZ rank (draw order)
Motorway / trunk14 m5
Primary11 m4
Secondary8 m3
Residential / unclassified5 m2
Footway / path1.5 m1
Railway3 m6

When Conform to Terrain is on, the road centerline is densified to ~6 m segments before draping so it follows hills smoothly. Roads ride ~0.12 m above the ground surface to prevent z-fighting with the terrain mesh.

07 Buildings

Placement algorithm

CityMaker's placement engine (ported from ObjPlaCr) uses:

  • Weighted-random selection — assets with more variety are chosen proportionally
  • Anti-repetition cooldown — recently-used variants are penalised
  • Size affinity — assets closest to the plot footprint are preferred
  • Deterministic seeding — same OSM file = same city every run (per OSM ID)
  • Serpentine (boustrophedon) ordering — processes plots spatially so cooldown works geographically

Building modes

Library Assets (Phase 1)
Place your FBX/OBJ/USD building models from the asset library, normalised to OSM footprint size.
Dummy City (Preview)
Procedural hash-coloured extruded polygons. No assets needed. Great for checking OSM data and layout before you add a library.

Individual Objects toggle

In dummy / preview mode, turning on Individual Objects makes each building, lamp, pole, and road a separate, selectable Blender object — grouped in CityMaker_Diagnostic/<category> sub-collections. Turn off for faster performance (one batched mesh per category).

08 Signboards New in 0.1

Signs are mounted on the placed building instances — CityMaker reads the real asset dimensions from the template collection so signs land exactly on the wall.

Where settingBehaviour
Random walls & cornersOne random wall or corner per building (varied across the city)
Front (longest wall)The longest facade — approximates the street-facing wall
Side wallsThe two shorter walls only
CornersAt building corners with diagonal outward normals
All wallsEvery wall above the minimum length threshold
Height settingBehaviour
Random heightVaries from 15% to 85% up the building wall per sign
Near top85% up — rooftop-style placement
Middle50% up — mid-wall
Near bottom18% up — shopfront level
Assets needed Create a Signboards/ subfolder in your asset library folder and add sign FBX/OBJ files there. Signs are normalised to ~3 m regardless of building size. Run Generate Buildings before Generate Signboards.

09 Export → Unreal Engine 5

Two different path types The Export Folder is a real folder on your hard drive (has a file picker). The UE Import Path is an in-engine virtual path like /Game/CityMaker/ — it is NOT a disk folder, it has no file picker. FBX never goes directly into your Content folder: Unreal's import script reads the FBX and creates the .uasset files under the Import Path.
  1. Set Export Folder (disk path)
    Click the folder icon next to Export Folder. Choose any folder outside your UE project — e.g. D:\UE_Export\CityMaker\. CityMaker writes FBX variants, CityMaker.json, and CityMaker_Import.py here.
  2. Set UE Import Path (in-engine)
    Type a Content Browser path like /Game/CityMaker/. This is where Unreal will create the .uasset files after import. No file picker — type it manually.
  3. Click Export
    CityMaker writes all files to the Export Folder. Each unique building variant becomes one FBX (not one per instance).
  4. Enable Python in Unreal
    Edit ▸ Plugins ▸ search "Python" ▸ enable Python Editor Script Plugin → restart the editor.
  5. Run the import script in Unreal
    File ▸ Execute Python Script… → browse to CityMaker_Import.py in the Export Folder → OK. The script imports the FBX (creating .uasset), enables Nanite, and places all buildings as Instanced Static Meshes from the JSON.
  6. Verify in Unreal
    Content Browser → your Import Path → you should see .uasset files. In the viewport and World Outliner, buildings appear grouped under CityMaker_Buildings.

Using assets already in Unreal (Synty / Fab / marketplace)

Export with Export Meshes (FBX) OFF — you get only CityMaker.json and CityMaker_Import.py. Open the script and edit the ASSET_REMAP dictionary to point variant names to your existing asset paths:

# CityMaker_Import.py — top of file
ASSET_REMAP = {
    "BLD_001": "/Game/Synty/PolygonCity/Meshes/SM_Bld_City_01",
    "BLD_002": "/Game/Synty/PolygonCity/Meshes/SM_Bld_City_02",
}

10 Export → Unity

  1. Export from Blender
    Set Target → Unity in the Export panel. Set the Export Folder to a folder outside your Unity project's Assets folder. Click Export. Blender writes FBX variants, CityMaker.json, and CityMakerImport.cs.
  2. Add FBX to Unity
    Drag the FBX files into your Unity Assets/CityMaker/Meshes/ folder. Unity auto-imports them. Set Scale Factor: 1 (Blender exports in metres). Create a Prefab from each mesh.
  3. Add the import script
    Copy CityMakerImport.cs to Assets/Editor/CityMaker/ in your Unity project. It must be under an Editor/ folder so it is excluded from game builds.
  4. Run the importer
    In Unity: Tools ▸ CityMaker ▸ Import City… Browse to CityMaker.json. Set the Asset Root to where your prefabs are. Click Import. Buildings appear in the scene hierarchy under CityMaker_Buildings.
Coordinate fix Blender (right-handed) and Unity (left-handed) have different coordinate systems. The import script applies a 90° X-rotation automatically. If buildings appear rotated 90°, check that coordinateFix = true in the importer.

11 CityMaker JSON

The cross-engine contract — all placement data in one portable file.

{
  "meta": { "origin_lat": 28.634, "origin_lon": 77.215, "area_km2": 5.73 },
  "placements": [
    {
      "variant": "BLD_002",
      "position": [-254.3, 178.3, -3.9],
      "rotation_z": 1.84,
      "scale": [1.15, 1.15, 1.15],
      "category": "buildings",
      "osm_id": "way/12345678"
    }
  ],
  "roads": [ ... ],
  "scatter": [ ... ]
}

Use the JSON-only export when assets are already in your engine. Any engine or tool that can read JSON can reconstruct the city without depending on Blender.

12 Live Test Scripts (Scripting Console)

What the Blender Scripting workspace is

Press Scripting tab at the top of Blender (or File ▸ New ▸ Scripting). The large panel on the left is a text editor where you open and run Python scripts. The 3D viewport shows changes in real-time.

How to run a live script

  1. Open the script
    In the Text Editor, click Open → navigate to D:\_deCode10\PluginWorks\CityMaker\tests\live\ → pick a script.
  2. Run it
    Click ▶ Run Script (top-right of the Text Editor) or press Alt+P.
  3. Watch the city build
    Each step prints progress and redraws the viewport. Full pipeline takes 15–60 s.

14 Troubleshooting

ProblemFix
Nothing gets placed / 0 buildingsTurn on Normalize Asset Size. Assets may be vastly different scales.
Terrain stays flat (no elevation)Needs internet + OSM <bounds>. Switch to Flat mode for offline work.
Buildings float above / sink into terrainClick Drape on Terrain. Make sure Conform to Terrain is on.
Roads don't follow terrainRoads are densified before draping. Rebuild roads after terrain: click Drape on Terrain which also rebuilds roads.
Signboards floating off buildingsSigns read the placed asset's real dimensions from the template collection. Make sure buildings were generated with Library Assets (not Dummy) mode.
ModuleNotFoundError: _live_commonEdit LIVE_DIR in tests/live/_live_common.py to match your repo path.
UE: Python not foundEnable Python Editor Script Plugin in UE Plugins → restart editor.
UE: buildings at wrong scaleThe import script converts Blender metres to UE centimetres. Check Output Log for scale warnings.
Unity: buildings rotated 90°Ensure coordinateFix = true in CityMakerImport.cs.
Viewport is slowHide finished collections in the Outliner. Use batched dummy mode (Individual Objects OFF) for very large cities.
