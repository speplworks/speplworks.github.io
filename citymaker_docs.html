Speaking Pictures · CityMaker Documentation

Build cities.
Your assets. Your look.

CityMaker turns OpenStreetMap data into a 3D city in Blender — preview it instantly, replace it with your own models, color and light it, and export to Unreal Engine 5 or Unity.

Blender 4.2+ OSM · SRTM UE5 · Unity · JSON 28 Categories

01 Quick Start

From a real place to a city in a few minutes.

  1. Install & open. Preferences ▸ Add-ons ▸ Install from Disk → pick the zip → enable. Press N in the viewport → CityMaker tab.
  2. Source. Load a .osm file (or set lat/lon and Download) and tick the categories you want. Operators auto-import the file, so a manual Import is optional.
  3. Preview. In Build City, tick Dummy Buildings and click Build Dummy City.
  4. Color it. Under Source ▸ Theme, choose a theme (e.g. Pastel City) and click Recolor City.
  5. Replace. Set a Library Folder, Refresh Library, then in Manipulate Your City click Replace Selected.
  6. Light & export. Lighting → Night → Apply → Add City Lights. Then Export → Unreal Engine 5.
No assets yet? The Dummy City + color themes is a complete look on its own — great for previs, drone sims, and stylized scenes.

03 Source (OSM)

Load a local .osm file, or enter latitude / longitude / area size and Download from Map (Overpass). Click Import / Refresh OSM to parse it. The OSM Categories list (top of the section) shows every category found — tick what to include.

04 Build City + Colors

Build City creates the structural preview. Tick Dummy Buildings (and optionally Individual Objects for one selectable object per feature), then Build Dummy City.

Theme colors (Source ▸ Theme)

Open the collapsible Theme panel under Source and choose a color mode (natural areas keep semantic green/blue automatically):

  • Theme Palette — built-in themes (Pastel City, Cyberpunk Night, Sunset, Concrete, Mono, Default) plus your own from a JSON file. Swatches are editable.
  • Random Pastel / Random Vivid — automatic per-building colors.
  • By Category — the classic hash coloring.

Recolor City re-applies colors without rebuilding; Randomize shuffles the seed. For varied per-building colors, use Individual Objects.

// citymaker_themes.sample.json — point "Themes File" at your own
{ "My Brand": [[0.05,0.45,0.85],[0.95,0.75,0.10],[0.20,0.20,0.25]] }

05 Library

Tell CityMaker where your models are. Folder mode expects one subfolder per category:

MyLibrary/
  buildings/    tower_a.fbx, house_01.glb, ...
  shops/        shop_a.fbx, ...
  street_lamps/ lamp_01.fbx, ...
  signboards/   sign_a.fbx, ...

Click Refresh Library — found groups show as green pills. Tagged Collections (mark objects from a bought .blend) and Another .blend are also supported. In Asset Options, keep Normalize Asset Size on for mixed-scale libraries.

06 Replace Assets

The core of CityMaker — swap any category for your own models.

In Manipulate Your City, each enabled category gets a row with a dropdown of your scanned library groups. Leave it on Auto (match name) to use the same-named group, or pick any group to override (you can even put lamps on buildings). Click Replace Selected — CityMaker decides footprint vs point placement per category, so you never have to.

  • Weighted placement with anti-repetition so the city never looks tiled.
  • Deterministic — the same city reproduces.
  • Idempotent — re-replacing a category replaces only that category.
Tip Tick Dry Run to see the plan in the system console before committing.

07 Spawner

Scatter any group — signboards, lights, decals, props — on/near placed objects. Pick the group, the category to spawn on, and a mode:

ModeBehaviour
Near (x/y/z offset)One per anchor at a fixed offset — any category
Front wallLongest wall ≈ street-facing (buildings)
Side walls / Corners / All wallsFaçade placement (buildings)
Random wall/cornerA random spot per building

Works on dummy buildings too (best with Individual Objects). Different spawn groups coexist — spawn lights and signs without one clearing the other.

08 Lighting

Pick a preset — Day / Sunset / Overcast / Night — and click Apply to set Blender's world and sun in one step.

City Lights (point lamps)

Add City Lights places point lamps. Pick a Layout:

  • Along Roads / Lines — street-lamp style: lamps at a set Spacing along roads, one or both sides, with a Main Roads Only filter and automatic junction de-clutter.
  • On Features — at a chosen category's objects.
  • Grid over Area — a regular grid covering everywhere.
  • Reach — how far one light illuminates (its lit pool).
  • Area Radius — light just one district around the selected object or 3D cursor. It's additive: light areas one by one without erasing earlier ones (0 = whole map).
  • Max Lights (default 200) and Cast Shadows (off for many — shadows are the slow part).

Emissive (lit-from-above look)

Tick By Emissive Material, pick a category (roads, shops, buildings) and a colour, then Apply Emissive — the objects themselves glow, so the city reads lit at any zoom. This is how you get a continuous glowing street network from above. Clear Emissive turns it off.

Visibility Lights/glow show in Material Preview / Rendered shading, not Solid. Cycles gives the best glowing-street result. Keep Max Lights modest and Shadows off for performance.

09 Terrain

Terrain is optional and built first (Build City ▸ 1 · Terrain) so the city — including the dummy — drapes onto it. Three sources:

  • Generate (SRTM) — real elevation for the area (needs internet; smoothed/subdivided). Want flat ground? Just skip terrain — placements sit at Z 0.
  • Heightmap Image — load a grayscale PNG/EXR; CityMaker builds a smooth displaced terrain. Height = relief in metres (for an EXR already in metres, set 1); Detail = grid resolution / polygon count.
  • Scene Object — pick any mesh (your sculpt, a DEM, a fictional landscape); CityMaker drapes onto it without modifying it.

Keep Drape Placements on Terrain on so everything follows the ground. Build terrain first, or build it after — the city re-drapes either way.

10 Roads

By default roads build as flat ribbon meshes at real widths (with sidewalks). Tick Roads as Curves to instead generate lightweight Blender curve objects:

  • One poly curve per road, carrying cm_road_width and cm_road_type.
  • Sweep a profile (Curve bevel) or instance your own modular road mesh along them with Geometry Nodes.
  • Far lighter than meshes — map them to your road assets.
Note Intersections are currently overlapping ribbons/curves (no merged junction geometry yet).

11 Export

Unreal Engine 5

  1. Set an Export Folder (a real disk folder) and a UE Import Path (an in-engine path like /Game/CityMaker).
  2. Click Export — CityMaker writes FBX variants, CityMaker.json and CityMaker_Import.py into the folder.
  3. In Unreal: File ▸ Execute Python Script → pick CityMaker_Import.py. It imports the meshes, enables Nanite, and places everything as Instanced Static Meshes — thousands of buildings in a few draw calls.

Unity / JSON

Choose Unity or CityMaker JSON to write the cross-engine contract (projection, every placement, every road). Phase 1 has no bundled C# importer — drive your own Unity placement from the JSON.

{
  "meta": { "origin_lat": 28.634, "origin_lon": 77.215 },
  "placements": [ { "variant":"BLD_002", "position":[-254,178,0], "rotation_z":1.84 } ],
  "roads": [ ... ]
}
